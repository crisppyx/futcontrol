# Sprint Backlog - Sprint 1

## User Stories

### User Story 1
*As a* administrador del sistema,
*I want* registrar equipos de fútbol,
*So that* pueda almacenar su información en la base de datos.

### User Story 2
*As a* administrador del sistema,
*I want* consultar la lista de equipos,
*So that* pueda visualizar la información registrada.

### User Story 3
*As a* administrador del sistema,
*I want* actualizar y eliminar equipos,
*So that* la información permanezca correcta y actualizada.

---

# Tareas

## Registrar equipos
- Crear el formulario de registro.
- Validar los campos obligatorios.
- Guardar la información en la base de datos.
- Mostrar un mensaje de registro exitoso.

## Consultar equipos
- Crear la vista para listar equipos.
- Mostrar todos los registros almacenados.
- Permitir la búsqueda de equipos por nombre.

## Actualizar y eliminar equipos
- Implementar la edición de datos.
- Validar la información antes de actualizar.
- Agregar la opción para eliminar equipos.
- Mostrar mensajes de confirmación o error.

---

# Impedimentos y Dependencias

## Impedimentos
- Problemas de conexión con la base de datos.
- Datos incompletos ingresados por el usuario.
- Errores en la validación de los formularios.

## Dependencias
- Tener creada la base de datos.
- Contar con conexión al servidor.
- Tener configurado el entorno de desarrollo.
- Disponer de permisos para modificar los registros.

---

# Definition of Done (DoD)

- Todas las historias de usuario fueron implementadas.
- Los equipos pueden registrarse correctamente.
- Los equipos pueden consultarse en una lista.
- La información puede actualizarse sin errores.
- Los equipos pueden eliminarse correctamente.
- Las validaciones funcionan adecuadamente.
- No existen errores críticos durante las pruebas.
- Los cambios fueron guardados correctamente en el repositorio.
