## Requisitos funcionales

- El sistema debe permitir el registro de nuevos usuarios.
- El sistema debe permitir el inicio de sesión mediante usuario y contraseña.
- El sistema debe permitir registrar equipos de fútbol.
- El sistema debe permitir consultar la lista de equipos registrados.
- El sistema debe permitir actualizar la información de un equipo.
- El sistema debe permitir eliminar equipos de la base de datos.
- El sistema debe validar que los campos obligatorios estén completos antes de guardar la información.
- El sistema debe mostrar mensajes de éxito o error según la operación realizada.
- ## Requisitos no funcionales

- El sistema debe ser fácil de usar e intuitivo.
- El tiempo de respuesta para las operaciones no debe superar los 2 segundos.
- La información almacenada debe mantenerse segura y sin pérdidas.
- El sistema debe estar disponible mientras exista conexión al servidor o a la base de datos.
- La interfaz debe ser clara y organizada para facilitar su uso.

---

## Stack Tecnológico

- *Lenguaje de programación:* Dart/flutter
- *Interfaz gráfica:* Java Swing
- *Base de datos:* firebase
- *Gestor de base de datos:* phpMyAdmin o MySQL Workbench
- *Control de versiones:* Git y GitHub
- *Entorno de desarrollo (IDE):* Visual Studio Code o NetBeans
